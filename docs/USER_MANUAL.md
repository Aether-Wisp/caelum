# Caelum 平台操作手册 (User Manual)

## 📖 1. 系统初始化与登录

### 1.1 访问 Web 界面

确保您已通过 `docker-compose up -d --build` 成功启动系统后台服务。
打开现代浏览器（推荐 Chrome 或 Edge），在地址栏输入：

> [http://localhost:8888](http://localhost:8888)

进入平台主页。您将看到系统的现代化仪表盘，展示当前系统状态、内存负载、工具依赖加载情况及历史分析任务。

### 1.2 配置 AI 引擎 (首次使用必备)

为了使 Caelum 拥有“决策大脑”，您必须在初次进入时配置对应大语言模型的 API 连接。

1. 在系统侧边栏或顶部导航找到 **[设置/配置 (Settings)]** 面板。
2. 在 **模型选择 (AI Model)** 下拉框中选取您要使用的服务商（如 OpenAI 的 GPT-4、基于 DeepSeek 的接口等）。
3. 填入对应的 **API Key**（如果您部署的是本地专属大模型 API，请将 Base URL 修改为您的本地模型地址）。
4. 点击 **保存配置 (Save)**。系统验证连接成功后即可使用。

---

## 🎯 2. 发起渗透测试任务

### 2.1 设定目标资产

返回 **[仪表盘 / 新建任务 (Dashboard / New Task)]**，找到目标输入栏：

- **单点 IP 测试**：直接填入目标地址，例如 `192.168.1.100` 或对应公网 IP。
- **Web 站点测试**：填入完整的 URL，例如 `http://test-target.local:8080/`。
- **网段发现扫描**：填入 CIDR 格式，例如 `10.0.0.0/24` _(提示：网段范围过大可能致使信息整理及 AI 总结耗时较长)_。

### 2.2 启动渗透流程

点击 **[开始渗透 (Start Pentest)]** 引擎即刻启动。Caelum 会将流程从最初的“自动化网络侦察”、“端口探活”，无缝推进至“漏洞识别”和“自动化武器利用”。

---

## 🔍 3. 实时监控与大模型交互控制

### 3.1 攻击链可视化跟踪

测试开始后，操作界面会跳转至 **任务实时详情页 (Task Live Details)**。这里被划分为几个核心监控区，让您随时掌控系统动向：

1.  **AI 思维链 (AI Thoughts / CoT)**：实时展示大语言模型的思考与决策推理边界。例如：“_发现目标开放了 6379 端口 (Redis)，存在未授权访问可能，接下来我将指导调用 redis-cli 进行探测或加载 ssh-keygen 尝试公钥写入..._”。
2.  **工具底层回显 (Terminal Output)**：实时捕获并回放底层 Kali Linux 环境工具（如 Nmap 扫描状态、SQLMap 跑表进度）的真实命令行输出，不遗漏任何细节。
3.  **发现资产表**：面板动态自动梳理和追加目前探测到的“端口字典”、“CVE漏洞”、“存活链接”等。

### 3.2 人工介入对话 (Human-in-the-Loop)

除了让它“自由开火”外，您也能在中途纠正 AI 的判断方向。
您可在平台的对话交互框中及时打断或提供引导指令：

> _"跳过 80 端口的扫描，我觉得 8080 端口上的 Tomcat 有 RCE 风险，直接帮我尝试弱口令或上传 war 包。"_

通过 MCP 协议，大模型将立刻停止当前子例程并遵从您的自然语言指挥调配新的一批底层脚本验证。

## 🚇 4. 高级进阶：内网多层穿透操作 (Bugku PAR 等场景)

Caelum 系统的一个极为核心的设计就是**原生支持横向多节点代理（Pivoting）**。当打下第一层边界节点后，针对内部私有网段（如 `10.10.x.x`）的操作非常简单：

1.  **全自动隧道构建**：您完全不需要手动配置繁琐的路由表。AI 在验证当前边界靶机可控后，会自动通过内存加载或无文件落地的方式向靶机注入 `FRP` 或 `Chisel`。
2.  **自动端口池协商**：Caelum 的 Session Manager 会自主在 10000-60000 间找到本地空闲端口作为监听桥。
3.  **内网透明扫描**：您此时可以通过交互框告诉 AI："_继续对 `10.10.10.1/24` 网段进行纵向渗透_"。系统的所有后台扫描与利用工具，都会在底层被无缝打上 `proxychains4 -q` 的代理钩子，像在攻击外网一样顺畅地进行内网探测。

## 📊 5. 战果审阅与渗透报告导出

当系统判定没有更多可横向路径或您点击 **[终止任务 (End Task)]** 后，即可获取总结陈述：

1. 点击界面的 **[生成最终报告 (Generate Final Report)]**。
2. Caelum 借由大模型的归纳总结能力，会根据这一路的真实数据为您制作一份格式美观的 Markdown/PDF 渗透测试报告。
3. **报告包含的绝佳素材内容**：
   - **直观风险图表**：将漏洞根据危险等级（严重、高危、中危）打上 CVSS 分数及色块标识。
   - **POC 证明**：利用链完整回放，AI 将写明“我们如何利用了这个洞”，提供核心利用命令记录。
   - **修复方案清单**：给出基于企业级的配置修补方案，不仅“管杀”并且“管埋”，直接切中大创赛评审标准中极为关注的“防御建设”部分。

## 💡 6. 常见异常排查指南

- **网页加载失败 / Docker 报错**：重启 Docker 桌面端，确认您的主机未被其他应用占用 `8888` 端口。如果需要看底层报错日志，在宿主机执行 `docker logs caelum-app -f`。
- **AI 拒绝执行敏感渗透（Safety 顾虑）**：如果是由于部分商用模型（如基础版 GPT 等）内置安全风控导致的拒绝，推荐选用专门针对漏洞研究测试优化的越狱接口，或者更换无严格过滤限制的服务商密钥配置。
- **Nmap 等扫描未反馈（结果为空）**：确认您在使用 Docker Compose 部署时，配置文件未修改，系统必须运行在特权模式（`privileged: true`）且采用共享主机网络网络（`network_mode: "host"`）才能正确下放底层通讯能力（Raw sockets）。
